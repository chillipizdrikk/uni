# Звіт по практичному завданню: Методи організації сценаріїв

Це шаблон-звіт, який заповніть за результатами виконання завдань. Додайте сюди знятки екрана, протоколи виконання (логи командної консолі), список файлів у папці та пояснення, як відтворити ваші результати на іншому комп'ютері.

1. Опис середовища
- Операційна система: Windows (версія)
- Версія Python: наприклад Python 3.11 (вказати шлях до інтерпретатора)
- Використані утиліти/компілятори: g++ (для C++), MSVC, інші (вказати версії)

2. Перелік файлів, надісланих як приклад:
- task1_open_associated.py
- task2_exec_single.py
- task2_exec_multiple.py
- task2_startfile_multi.py
- task3_wait_main.py
- task3_child.py
- task3_async_main.py
- task3_async_child.py
- task3_cpp_child.cpp
- (додайте знімки екрану та логи)

3. Інструкція запуску (по кроках)
- Для Task 1:
  1. Скопіюйте в робочу теку один Word-файл (.docx/.doc) і один файл зображення (.jpg/.jpeg/.bmp).
  2. Запустіть task1_open_associated.py (через консоль або подвійний клік).
  3. Скрипт відкриє пов'язані аплікації. Зробіть 1–2 редагування вручну та закрийте їх.
  4. Поверніться до консолі й натисніть Enter, щоб завершити скрипт.

- Для Task 2:
  - Варіант a (передача одного файлу через exec):
    1. Скопіюйте у теку декілька .txt файлів.
    2. Запустіть task2_exec_single.py — він обере перший файл і замінить процес Python на notepad.exe.
    3. Після закриття notepad.exe процес завершиться (припиниться інтерпретатор).
  - Варіант b (передача списку файлів):
    1. Скопіюйте у теку декілька файлів (наприклад .jpg).
    2. Запустіть task2_exec_multiple.py, вкажіть розширення (jpg) і дочекайтесь виконання os.execvp.
    3. Якщо ваша цільова програма підтримує множинні файли (наприклад mspaint), вона відкриє всі передані файли.
  - Альтернатива:
    - task2_startfile_multi.py — відкриває всі знайдені файли у пов'язаних програмах за допомогою os.startfile (асинхронно).

- Для Task 3:
  - Сценарій очікування дочірнього процесу:
    1. Переконайтесь, що у теці є task3_child.py.
    2. Запустіть task3_wait_main.py — він запише початкові дані у comdata.txt, викличе дочірній скрипт, дочірній модифікує дані й запише назад.
    3. Після завершення дочірнього процесу основний прочитає оновлені дані, додасть 1 до кожного елемента і знову запише файл.
  - Сценарій асинхронного виконання:
    1. Запустіть task3_async_main.py (в окремій консолі). Він створить паралельний дочірній процес task3_async_child.py за допомогою Popen.
    2. Основний процес продовжить роботу, перевірить стан дочірнього процесу через poll().

4. C++ дочірній процес
- Файл: task3_cpp_child.cpp
- Призначення: приклад простої програми, що читає comdata.txt, множить числа на 10 та записує назад.
- Компіляція:
  - Використовуючи g++: g++ -O2 -std=c++17 task3_cpp_child.cpp -o task3_cpp_child.exe
  - АБО MSVC (Developer Command Prompt): cl /EHsc task3_cpp_child.cpp
- Запуск з Python:
  - У task3_wait_main.py можна замінити виклик до Python-дочірнього процесу
    на subprocess.run(["task3_cpp_child.exe"]) (перед тим переконатись, що exe у тій же папці).

5. Протоколи виконання (логи)

  - Вивід task3_wait_main.py до/після запуску дочірнього процесу.
  - Вивід task3_child.py.
  - Вивід task3_async_main.py і task3_async_child.py.
- Додайте знімки екрана: списки файлів у файловому менеджері, відкрите вікно Word/переглядача зображень, зображення структури папок.

6. Примітки та попередження
- os.startfile працює тільки у Windows.
- os.exec* повністю замінює поточний процес — код після виклику не виконається.
- Popen дозволяє асинхронне виконання, run() очікує завершення дочірнього процесу.
- Будьте обережні при одночасному відкритті великої кількості файлів (можливі відмови аплікацій).
