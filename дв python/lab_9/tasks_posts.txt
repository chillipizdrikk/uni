https://jsonplaceholder.typicode.com/posts
Опис: список тестових “постів” (userId, id, title, body). Для демонстрацій структури масиву об’єктів.

Задачі для аналізу файлу posts.json.

Коротко про структуру:
- Файл містить список об'єктів (постів).
- Кожен пост має поля: userId (ціле), id (ціле), title (рядок), body (рядок).

Задача 1 - Топ користувачів за кількістю постів
Опис: Порахувати кількість постів для кожного userId. Повернути топ‑5 користувачів із найбільшою кількістю постів у форматі:
  userId, post_count, list_of_post_ids (відсортований).
Якщо кількість користувачів менша за 5, вивести всіх.

Задача 2 - Статистика довжини заголовків і тіл постів
Опис: Для кожного userId обчислити:
  - середню довжину title (у символах),
  - середню довжину body (у символах).
Повернути userId -> {avg_title_len, avg_body_len}, відсортувати за userId.
Крім того, вивести загальний список топ‑10 постів із найбільшими body (id, userId, body_length, title).

Задача 3 - Частотний аналіз слів і пошук за ключовими словами
Опис:
  - Побудувати частотний словник слів (токенізація простим розбиттям; регістр і пунктуація ігноруються) по полях title+body для всіх постів.
  - Вивести топ‑20 найчастіших слів (окрім простих стоп‑слів: a, the, and, in, of, to, is, it, for, on, that, this).
  - Для заданого списку ключових слів (наприклад: ["qui", "et", "sunt"]) вивести для кожного слова перелік id постів, де це слово зустрічається (регістронезалежний, слово як окремий токен).
